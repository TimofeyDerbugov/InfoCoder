№1 ОГЭ по информатике

Кодирование информации. Размер текста в байтах, кодировка Unicode

Чаще всего встречаются задания на поиск лишнего слова в предложении. Даны размеры одного символа и вес файла, на который уменьшился начальный вес файла. Для решения понадобится таблица перевода единиц измерения.

Пользуясь этой таблицей, решаются все типы заданий № 1.

Алгоритм решения задач

  • Перевести все числа в одну единицу измерения.
  • Найти количество лишних символов.
  • Из полученного количества символов вычесть 2 символа (пробел и запятая тоже считаются как 1 символ).
  • Найти слово по количеству лишних символов.

Отлично! Теперь давайте потренируемся на реальных заданиях из ОГЭ.

Пример

Вова записал текст в документе, где используется шестнадцатибитная кодировка (1 символ = 16 бит). В тексте нет лишних пробелов. У него получилось:

«Устройства ввода информации: клавиатура, мышь, сканер, тачпад, планшет, шар, джойстик».

Затем он добавил в текст название ещё одного устройства — «микрофон», а также пробел и запятую. Вычислите, на сколько байтов увеличился получившийся текст в данной кодировке.

Напишите в ответе количество байтов (единицы измерения писать не нужно).

Решение

  1. Определяем нужные данные:
    • шестнадцатибитная кодировка;
    • добавил «микрофон», а также пробел и запятую;
    • нужно найти, на сколько байтов увеличился текст;
    • ответ должен быть в байтах.
  2. Определяем вес 1 символа:
    • шестнадцатибитная кодировка ⇒ 1 символ = 16 бит;
    • 16 бит = 2 байта (вес одного символа в байтах);
    • «микрофон» = 8 символов;
    • 8 символов × 2 байта = 16 байт.

Ответ: 16

Задание для отработки № 1

Задача:
В кодировке КОИ-8 каждый символ кодируется 8 битами. Аня написала текст (в нём нет лишних пробелов):
«Ёрш, щука, бычок, карась, гимнура, долгопёр — рыбы».
Она вычеркнула из списка название одной рыбы, а также лишние запятую и пробел. При этом размер нового предложения оказался на 10 байт меньше, чем размер исходного. Напишите в ответе вычеркнутое название рыбы.

Решение:
1. Вес одного символа: 8 бит = 1 байт.
2. Размер уменьшился на 10 байт ⇒ удалено 10 символов.
3. Из них 2 символа — запятая и пробел.
4. Значит, название рыбы состоит из 10 − 2 = 8 символов.
5. Проверим слова:
- «Ёрш» — 3 символа;
- «щука» — 4 символа;
- «бычок» — 5 символов;
- «карась» — 6 символов;
- «гимнура» — 7 символов;
- «долгопёр» — 8 символов.
6. Подходит только «долгопёр».
Ответ: долгопёр

Задание для отработки № 2

Задача:
В Unicode каждый символ кодируется 2 байтами. Петя написал текст (без лишних пробелов):
«Собака, кошка, курица, корова, лошадь, коза, овца — домашние животные».
Он вычеркнул одно название животного, а также удалённые запятую и пробел. Новый размер текста оказался на 16 байт меньше исходного. Какое животное вычеркнул Петя?

Решение:
1. Вес одного символа — 2 байта.
2. Размер уменьшился на 16 байт ⇒ удалено 16 ÷ 2 = 8 символов.
3. Из них 2 символа — запятая и пробел.
4. Значит, название животного — 8 − 2 = 6 символов.
5. Проверяем слова:
- «Собака» — 5 символов;
- «кошка» — 5 символов;
- «курица» — 6 символов;
- «корова» — 6 символов;
- «лошадь» — 6 символов;
- «коза» — 3 символа;
- «овца» — 4 символа.
6. Подходят «курица», «корова», «лошадь». Но в предложении после вычёркивания не должно быть двух пробелов подряд — значит, удалено слово не в начале и не в конце списка (иначе остался бы один пробел). Ищем слово в середине: «курица» или «корова».
7. В условии не указано, что ответ единственный, но обычно выбирают первое подходящее слово из списка — «курица».
Ответ: курица

№2 ОГЭ по информатике

Двоичный код, шифровка. Распознавание символов по коду Морзе.

Задание № 2 ОГЭ по информатике проверяет умение работать с кодами и шифрами. Ученику дают сообщение, закодированное с помощью таблицы символов (например, азбуки Морзе или собственного шифра), и просят либо декодировать его, либо подобрать правильную кодировку. Требуется внимательность к деталям, знание принципов кодирования и умение сопоставлять символы с их кодами.

Алгоритм решения

  1. Внимательно изучите предоставленную кодовую таблицу (соответствие букв и их кодов).
  2. Разбейте зашифрованную последовательность на отдельные коды, опираясь на длину кодов из таблицы.
  3. Сопоставьте каждый код с соответствующей буквой из таблицы.
  4. Запишите полученную последовательность букв — это и будет ответ.
  5. Проверьте, нет ли неоднозначных вариантов разбиения (если есть — выберите тот, который даёт осмысленное слово/фразу).

№3 ОГЭ по информатике

Логические высказывания. Работа с «и», «или», «не».

Задание № 3 ОГЭ по информатике посвящено логическим операциям и условиям. Ученику предлагают высказывание с операторами «И», «ИЛИ», «НЕ» и просят определить, при каких значениях переменных оно истинно или ложно. Нужно уметь разбивать сложное выражение на части, применять законы логики, анализировать условия и проверять варианты ответов. Задание проверяет навыки формального мышления и работы с булевой алгеброй на базовом уровне.

Алгоритм решения

  1. Внимательно прочитайте условие и выделите логическое выражение.
  2. Определите, при каких значениях выражение должно быть истинным/ложным (в зависимости от вопроса).
  3. Разбейте сложное выражение на простые части (операции И, ИЛИ, НЕ).
  4. Для каждой простой части определите условия истинности/ложности.
  5. Найдите пересечение условий (если есть операция И) или объединение (если ИЛИ).
  6. Проверьте все варианты ответа, подставляя значения в исходное выражение.
  7. Выберите правильный ответ, соответствующий условию задачи.

№4 ОГЭ по информатике

Задание № 4 ОГЭ по информатике: формальные описания реальных объектов и процессов (графы, длина путей)

Тема: Задание проверяет умение анализировать таблицы и строить графы для поиска кратчайшего пути между пунктами. Требуется определить минимальную длину маршрута с учётом заданных дорог.

Что нужно знать

  • Граф — набор вершин (пунктов) и рёбер (дорог между ними).
  • Взвешенный граф — рёбра имеют «вес» (длину дороги).
  • Путь в графе — последовательность вершин, где каждая следующая соединена с предыдущей ребром.
  • Длина пути — сумма весов рёбер, составляющих путь.

Алгоритм решения

  1. Постройте граф по данным таблицы: обозначьте вершины (пункты), соедините их рёбрами с указанием длины дорог.
  2. Выделите начальную и конечную вершины (пункты, между которыми ищем путь).
  3. Перечислите все возможные пути из начального пункта в конечный.
  4. Для каждого пути посчитайте общую длину (сумму весов рёбер).
  5. Выберите путь с минимальной длиной.
  6. Запишите ответ — число (длина кратчайшего пути).

Пример задания (типовой формат)

Задача:
Между населёнными пунктами A, B, C, D, E построены дороги, протяжённость которых (в км) приведена в таблице:

A B C D E
A - 2 4 6
B 2 - 1 4 7
C 4 1 - 5 1
D 4 5 - 3
E 6 7 1 3 -

Определите длину кратчайшего пути между пунктами A и D. Передвигаться можно только по указанным дорогам.

Решение

Построим возможные пути из A в D и посчитаем их длину:

  1. A → B → D: 2 (AB) + 4 (BD) = 6 км.
  2. A → B → C → D: 2 (AB) + 1 (BC) + 5 (CD) = 8 км.
  3. A → B → C → E → D: 2 + 1 + 1 (CE) + 3 (ED) = 7 км.
  4. A → C → D: 4 (AC) + 5 (CD) = 9 км.
  5. A → C → E → D: 4 + 1 (CE) + 3 = 8 км.
  6. A → E → D: 6 (AE) + 3 (ED) = 9 км.
  7. A → E → C → D: 6 + 1 + 5 = 12 км.

Сравниваем длины:

  • 6 км (A→B→D),
  • 8 км,
  • 7 км,
  • 9 км,
  • 12 км.

Наименьшая длина — 6 км.

Ответ: 6

№5 ОГЭ по информатике

Анализ алгоритмов

Тема: Задание проверяет умение анализировать простые алгоритмы с фиксированным набором команд. Ученику дают последовательность команд и числовое значение на входе; требуется вычислить результат или найти параметр (например, значение коэффициента), при котором выполняется заданное условие.

Алгоритм решения

  1. Внимательно прочитайте условие и выделите:
    • исходное число (или диапазон чисел);
    • набор команд алгоритма;
    • условие, которое должно выполняться в итоге (например, «результат делится на 5»).
  2. Выполните команды алгоритма пошагово для заданного числа.
  3. Если требуется найти параметр — подберите его методом перебора или составьте уравнение на основе итоговых условий.
  4. Проверьте полученный результат на соответствие условию задачи.
  5. Запишите ответ в требуемой форме (число, последовательность и т. п.).

Разбор типового задания

Задача:
У исполнителя «Вычислитель» две команды:
1. умножь на 3
2. прибавь 2
Программа для Вычислителя — это последовательность номеров команд. Например, 121 означает: умножь на 3, прибавь 2, умножь на 3.
Определите результат выполнения программы 121 для числа 1.

Решение

  1. Начальное число: 1.
  2. Команда 1 (умножь на 3): 1 × 3 = 3.
  3. Команда 2 (прибавь 2): 3 + 2 = 5.
  4. Команда 1 (умножь на 3): 5 × 3 = 15.
Ответ: 15

Задание для отработки № 1

Задача:
У исполнителя «Вычислитель» две команды:
1. умножь на 2
2. вычти 1
Известно, что программа переводит число 5 в число 17.
Определите последовательность команд программы. В ответе запишите только номера команд без пробелов и запятых.

Решение

Будем двигаться от конечного числа (17) к начальному (5), подбирая обратные операции:

  1. Число 17 нечётное → не могло получиться умножением на 2 → последняя команда была «вычти 1» (команда 2).
    Предыдущее число: 17 + 1 = 18.
  2. Число 18 чётное → могло получиться умножением на 2 → предпоследняя команда «умножь на 2» (команда 1).
    Предыдущее число: 18 ÷ 2 = 9.
  3. Число 9 нечётное → команда 2. Предыдущее число: 9 + 1 = 10.
  4. Число 10 чётное → команда 1. Предыдущее число: 10 ÷ 2 = 5.
  5. Достигли исходного числа 5 → цепочка завершена.

Запишем команды в прямом порядке (от 5 к 17):

  • 5 × 2 = 10 (команда 1)
  • 10 − 1 = 9 (команда 2)
  • 9 × 2 = 18 (команда 1)
  • 18 − 1 = 17 (команда 2)
Ответ: 1212

У исполнителя «Удвоитель» команды:

К числу 4 применили программу 1212. Каков результат?

№6 ОГЭ по информатике

Анализ программ с условиями

Тема: Задание проверяет умение анализировать короткие программы (или их описания) с условными операторами. Ученику дают код или словесное описание алгоритма с ветвлениями; требуется определить:

  • какой результат будет выведен при заданных входных данных;
  • при каких входных данных программа выведет заданный результат;
  • сколько раз программа выведет определённый ответ при серии запусков.

Задача часто включает параметр (например, переменную A), который нужно подобрать под условие.

Алгоритм решения

  1. Внимательно прочитайте условие и выделите:
    • входные переменные (обычно s, t, иногда A);
    • логическое условие (например, «если s > A или t > 12»);
    • выводимый результат («ДА» / «НЕТ», «YES» / «NO»).
  2. Для каждого тестового набора входных данных:
    • подставьте значения s и t в условие;
    • проверьте, выполняется ли условие (истина/ложь);
    • определите, какой результат выведет программа.
  3. Если нужно найти параметр A:
    • перебирайте целые значения A;
    • для каждого A проверьте все тестовые запуски;
    • подсчитайте, сколько раз выводится нужный результат;
    • выберите A, при котором количество совпадений соответствует условию задачи.
  4. Запишите ответ (число, количество запусков и т. п.).

Разбор типового задания

Задача:
Дана программа (на псевдокоде):
ввод s
ввод t
ввод A
если (s > A) или (t > 12)
вывод "YES"
иначе
вывод "NO"

Было проведено 9 запусков с парами (s, t):
(1, 2), (11, 2), (1, 12), (11, 12), (−11, −12), (−11, 12), (−12, 11), (10, 10), (10, 5).
Укажите целое значение параметра A, при котором программа выведет «NO» ровно 3 раза.

Решение

  1. Программа выводит «NO», когда оба условия ложны:
    • s ≤ A и
    • t ≤ 12.
  2. Анализируем пары (s, t) и условие t ≤ 12:
    • (1, 2): t = 2 ≤ 12 → возможно «NO»;
    • (11, 2): t = 2 ≤ 12 → возможно «NO»;
    • (1, 12): t = 12 ≤ 12 → возможно «NO»;
    • (11, 12): t = 12 ≤ 12 → возможно «NO»;
    • (−11, −12): t = −12 ≤ 12 → возможно «NO»;
    • (−11, 12): t = 12 ≤ 12 → возможно «NO»;
    • (−12, 11): t = 11 ≤ 12 → возможно «NO»;
    • (10, 10): t = 10 ≤ 12 → возможно «NO»;
    • (10, 5): t = 5 ≤ 12 → возможно «NO».
    → Все 9 пар проходят по t ≤ 12.
  3. Теперь подбираем A, чтобы ровно 3 пары дали s ≤ A:
    • Если A = 0: s ≤ 0 у пар (−11, −12), (−11, 12), (−12, 11) → 3 «NO».
    • Проверяем: для остальных 6 пар s > 0 → «YES».
  4. При A = 0 условие выполнено: 3 «NO» и 6 «YES».
Ответ: 0

Задание для отработки № 1

Задача:
Программа:
ввод s
ввод t
ввод A
если (s > 5) или (t < A)
вывод "ДА"
иначе
вывод "НЕТ"

Запуски с парами (s, t): (3, 4), (7, 2), (5, 8), (2, 3), (8, 1), (4, 6), (6, 5), (1, 7), (9, 4).
При каком наименьшем целом A программа выведет «ДА» ровно 5 раз?

Решение

  1. «ДА» выводится, если s > 5 или t < A.
  2. Считаем, для каких пар s > 5 (они дадут «ДА» независимо от A):
    • (7, 2): s = 7 > 5 → «ДА»;
    • (8, 1): s = 8 > 5 → «ДА»;
    • (6, 5): s = 6 > 5 → «ДА»;
    • (9, 4): s = 9 > 5 → «ДА».
    → Уже 4 запуска дают «ДА» только за счёт s > 5.
  3. Нужно, чтобы ровно 1 дополнительная пара дала «ДА» за счёт условия t < A (тогда всего будет 5 «ДА»).
  4. Рассмотрим оставшиеся пары (s ≤ 5) и их t:
    • (3, 4): t = 4;
    • (5, 8): t = 8;
    • (2, 3): t = 3;
    • (4, 6): t = 6;
    • (1, 7): t = 7.
  5. Подберём наименьшее A, при котором только одна из этих пар удовлетворяет t < A:
    • Если A = 4: t < 4 только у (2, 3) → 1 дополнительная «ДА».
    • Проверяем: при A = 4 всего «ДА»: 4 (по s) + 1 (по t) = 5.
    • При A = 3: ни одна пара не даст t < 3 → только 4 «ДА» (не подходит).
    • При A = 5: t < 5 у (3, 4) и (2, 3) → 2 дополнительные «ДА» → всего 6 «ДА» (не подходит).
  6. Значит, наименьшее подходящее A = 4.
Ответ: 4

№8 ОГЭ по информатике

Круги Эйлера (диаграммы множеств)

Задание проверяет умение работать с множествами и логическими операциями («И», «ИЛИ») через диаграммы Эйлера. Требуется находить количество элементов в областях пересечения множеств.

Основные обозначения и логика

  • A | B («A ИЛИ B») — объединение множеств: все элементы, принадлежащие A, B или обоим сразу.
  • A & B («A И B») — пересечение множеств: только элементы, принадлежащие одновременно A и B.

Базовые формулы

1. Для двух множеств (A и B)

Общая формула включений‑исключений:

|A | B| = |A| + |B| - |A & B|

Где:

  • |A | B| — количество элементов в объединении A и B («A ИЛИ B»);
  • |A| — количество элементов в множестве A;
  • |B| — количество элементов в множестве B;
  • |A & B| — количество элементов в пересечении A и B («A И B»).

Отсюда можно выразить пересечение:

|A & B| = |A| + |B| - |A | B|

2. Для трёх множеств (A, B и C)

Формула включений‑исключений для объединения трёх множеств:

|A | B | C| = |A| + |B| + |C| - |A & B| - |A & C| - |B & C| + |A & B & C|

Где:

  • |A | B | C| — «A ИЛИ B ИЛИ C» (все элементы хотя бы в одном множестве);
  • |A & B|, |A & C|, |B & C| — попарные пересечения («A И B», «A И C», «B И C»);
  • |A & B & C| — элементы, принадлежащие всем трём множествам одновременно.

Для нахождения отдельных областей на диаграмме часто составляют систему уравнений, опираясь на известные значения.

Алгоритм решения

  1. Определите, какие множества даны (A, B, C и т. д.).
  2. Выпишите известные величины из условия (например, |A|, |A | B|, |A & B|).
  3. Выберите подходящую формулу (для 2 или 3 множеств).
  4. Подставьте известные значения и решите уравнение.
  5. Если нужно найти конкретную область диаграммы — разбейте её на части и выразите через известные величины.

Пример задания (2 множества)

Задача:
В языке запросов поискового сервера:
- запрос Корабль нашёл 310 страниц;
- запрос Самолёт нашёл $x$ страниц;
- запрос Корабль | Самолёт нашёл 840 страниц;
- запрос Корабль & Самолёт нашёл 80 страниц.
Сколько страниц найдёт запрос Самолёт?

Решение

Используем формулу для двух множеств:

|A | B| = |A| + |B| - |A & B|

Подставляем известные значения:

840 = 310 + x - 80 840 = 230 + x x = 840 - 230 = 610 Ответ: 610

Пример задания (3 множества)

Задача:
В языке запросов:
- Футбол | Хоккей | Теннис — 1200 страниц;
- Футбол — 600;
- Хоккей — 500;
- Теннис — 400;
- Футбол & Хоккей — 200;
- Футбол & Теннис — 150;
- Хоккей & Теннис — 100.
Сколько страниц даст запрос Футбол & Хоккей & Теннис?

Решение

Применяем формулу для трёх множеств:

|A | B | C| = |A| + |B| + |C| - |A & B| - |A & C| - |B & C| + |A & B & C|

Подставляем значения:

1200 = 600 + 500 + 400 - 200 - 150 - 100 + x 1200 = 1500 - 450 + x 1200 = 1050 + x x = 1200 - 1050 = 150 Ответ: 150

Задание для отработки № 1 (2 множества)

Задача:
По запросам найдено:
- Яблоко — 450 страниц;
- Груша — 300 страниц;
- Яблоко | Груша — 600 страниц.
Сколько страниц даст запрос Яблоко & Груша?

Решение

|A & B| = |A| + |B| - |A | B| = 450 + 300 - 600 = 150 Ответ: 150

Задание для отработки № 2 (три множества)

Задача:
В языке запросов поискового сервера для обозначения логической операции «ИЛИ» используется символ «|», а для логической операции «И» — символ «&».

По запросу найдено следующее количество страниц:

  • Море & Пляж — 250 страниц;
  • Море & Отель — 300 страниц;
  • Пляж & Отель — 180 страниц;
  • Море & Пляж & Отель — 90 страниц;
  • Море | Пляж | Отель — 1 200 страниц.

Известно также, что:

  • по запросу Море найдено 600 страниц;
  • по запросу Пляж — 500 страниц.

Определите, сколько страниц будет найдено по запросу Отель.

Решение

Используем формулу включений‑исключений для трёх множеств:

|M | П & О| = |M| + |П| + |О| - |M & П| - |M & О| - |П & О| + |M & П & О|

Подставим известные значения:

  • |M | П | О| = 1200;
  • |M| = 600;
  • |П| = 500;
  • |M & П| = 250;
  • |M & О| = 300;
  • |П & О| = 180;
  • |M & П & О| = 90.

Обозначим |О| = x (искомое количество страниц по запросу «Отель»).

Составим уравнение:

1200 = 600 + 500 + x - 250 - 300 - 180 + 90

Упростим правую часть:

600 + 500 = 1100 1100 - 250 = 850 850 - 300 = 550 550 - 180 = 370 370 + 90 = 460 1200 = 460 + x

Найдём x:

x = 1200 - 460 = 740 Ответ: 740

Анализ схем и подсчёт путей

Задание проверяет умение анализировать схемы дорог (графов) и подсчитывать количество возможных путей между двумя пунктами с учётом направления движения.

Основные типы заданий

  1. Тип 1: поиск всех возможных путей из одного города в другой.
  2. Тип 2: поиск путей, проходящих или не проходящих через определённый пункт.

Алгоритм решения (общий подход)

  1. На схеме отмечаем начальный пункт (ставим число 1).
  2. Для каждого следующего пункта считаем сумму чисел всех «входящих» дорог (тех, что ведут в этот пункт).
  3. Если путь должен проходить через конкретный пункт — игнорируем дороги, не связанные с ним.
  4. Если путь не должен проходить через пункт — «закрываем» все дороги, ведущие в него и из него (считаем как 0).
  5. Продолжаем подсчёт до конечного пункта.

Пример задания (Тип 1: все пути)

Задача:
На рисунке — схема дорог, связывающих города А, Б, В, Г, Д, Е, Ж, З, И, К и Л. По каждой дороге можно двигаться только в направлении, указанном стрелкой. Сколько существует различных путей из города А в город Л?

Решение

Последовательно считаем количество путей до каждого города:

  • А = 1 (начальный пункт).
  • Б = 1 (из А).
  • В = А + Б = 1 + 1 = 2.
  • Г = Д + А = 1 + 1 = 2.
  • Д = 1 (из А).
  • Е = Б + В = 1 + 2 = 3.
  • Ж = Г + Д = 1 + 2 = 3.
  • З = Е + В + Г + Ж = 3 + 2 + 2 + 3 = 10
  • И = Е = 3 (только из Е).
  • К = Ж = 3.
  • Л =И + Е + З + Ж + К = 3 + 3 + 10 + 3 + 3 = 22.
Ответ: 22

При условии "не проходящие через город Е", отсекаем пути, идущие через эту точку.

При условии "Проходящие через город Е", отсекаем альтернативные пути (в случае сверху, это пути чере точки И, Д, Ж).

  2. Проанализируйте стартовую обстановку: где Робот, где стены, какие клетки нужно закрасить.
  3. Разбейте задачу на подзадачи (например: дойти до стены, повернуть, закрасить ряд и т. д.).
  4. Для каждой подзадачи напишите последовательность команд.
  5. Используйте циклы, если есть повторяющиеся действия.
  6. Проверьте, что Робот не врезается в стену (используйте проверки условий).
  7. Протестируйте алгоритм «в уме» на примере из условия.
  8. Запишите итоговое решение чётко и без ошибок.

Типичные ошибки (избегайте их!)

  • Робот «упирается» в стену (нет проверки условия перед движением).
  • Не закрашены нужные клетки или закрашены лишние.
  • Цикл не завершается (неверное условие).
  • Ошибки в синтаксисе (пропущены «нц», «кц», «все» и т. п.).
  • Лишние команды, усложняющие алгоритм.

Советы по оптимизации

  • Всегда начинайте с проверки условий перед движением.
  • Используйте циклы для повторяющихся действий — это сокращает код.
  • Если нужно закрасить «лестницу» или «зигзаг», подумайте, как описать это циклом с условием.
  • Проверяйте, что алгоритм работает для разных начальных положений Робота (если это допустимо по условию).

Два типа ввода данных

1. Последовательность с известной длиной (задано N)

n = int(input())
for i in range(n):
    a = int(input())
    # обработка числа a

Пример: сначала вводится количество чисел, затем сами числа.

2. Последовательность до нуля (неизвестная длина)

a = int(input())
while a != 0:
    # обработка числа a
    a = int(input())

Пример: числа вводятся до тех пор, пока не введён 0 (0 — признак конца).

Основные типы задач и шаблоны

Поиск суммы чисел по условию

sum_vals = 0
# ... ввод чисел ...
if условие_для_a:
    sum_vals += a
# ...
print(sum_vals)

Подсчёт количества чисел по условию

count = 0
# ... ввод чисел ...
if условие_для_a:
    count += 1
# ...
print(count)

Поиск максимума среди чисел по условию

max_num = 0  # или очень маленькое число
# ... ввод чисел ...
if условие_для_a and a > max_num:
    max_num = a
# ...
print(max_num)

Поиск минимума среди чисел по условию

min_num = 30001  # число больше всех возможных входных
# ... ввод чисел ...
if условие_для_a and a < min_num:
    min_num = a
# ...
print(min_num)

Среднее арифметическое (с проверкой на ноль)

sum_vals = 0
count = 0
# ... ввод чисел ...
if условие_для_a:
    sum_vals += a
    count += 1
# ...
if count > 0:
    print(f'{sum_vals / count:.1f}')  # 1 знак после запятой
else:
    print('NO')

Частые условия для проверки

  • a % 2 == 0 — число чётное
  • a % 2 != 0 — число нечётное
  • a % k == 0 — число делится на k без остатка
  • a % 10 == x — последняя цифра числа равна x
  • 10 <= a <= 99 — число двузначное
  • 100 <= a <= 999 — число трёхзначное
  • a > 0 — число положительное

Типичные ошибки

  1. Неправильная инициализация переменных (min_num слишком маленькое, max_num слишком большое).
  2. Деление на ноль при вычислении среднего (не проверена переменная count).
  3. Ошибки в условиях (например, a % 10 = 5 вместо a % 10 == 5).
  4. Пропуск ввода первого числа в цикле while.
  5. Вывод результата не в том формате (например, без «NO» при отсутствии подходящих чисел).

Пошаговый алгоритм решения

  1. Прочитайте условие задачи: что нужно найти (сумму, количество, максимум, минимум, среднее)?
  2. Определите тип ввода: известно N или ввод до нуля?
  3. Выберите шаблон (см. выше) под нужный тип задачи.
  4. Запишите условие проверки для чисел (например, «чётное и трёхзначное»).
  5. Добавьте ввод данных по выбранному шаблону.
  6. Внедрите условие проверки внутрь цикла.
  7. После цикла выведите результат с учётом особых случаев (например, «NO», если подходящих чисел нет).
  8. Проверьте код на примере из условия.
